# 2026-04-24

## emotion-detect-android 项目初始构建完成

从零创建 Android 人脸情绪检测 App：
- 技术：MediaPipe Face Landmarker (0.10.14) + CameraX (1.3.4) + Kotlin
- 核心文件：FaceLandmarkerHelper / EmotionClassifier / OverlayView / MainActivity
- 模型：face_landmarker.task（3.76MB，已下载到 assets/）
- 情绪：7种（Happy/Sad/Surprised/Angry/Disgusted/Fearful/Neutral）
- 注意：faceBlendshapes() 返回 Optional<List<List<Category>>>，需 .isPresent 判断
- 构建方式：用 Android Studio 打开根目录，连接真实设备运行（不支持模拟器）

## 补丁：gradle.properties 缺失
- 报错：`android.useAndroidX property is not enabled`
- 根因：项目构建时遗漏了 gradle.properties 文件
- 修复：创建 gradle.properties，添加 android.useAndroidX=true

## 补丁：OverlayView 颜色字符串非法
- 报错：`IllegalArgumentException: Unknown color` at OverlayView.kt:83
- 根因：faceOutlinePaint 的颜色字符串 `#6006200EE` 有9位，不合法
- 修复：改为合法8位 `#6006C20E`（#AARRGGBB 格式）

## 补丁：OverlayView 坐标映射错误导致轮廓偏小居中
- 现象：人脸轮廓只覆盖脸部中心区域，没有覆盖整张脸
- 根因：scaleX=viewW/imgW、scaleY=viewH/imgH 独立缩放破坏宽高比，坐标被压缩到中间
- 修复：改用 scale=min(viewW/imgW, viewH/imgH) 等比缩放 + offsetX/Y 居中偏移
- 同步：PreviewView scaleType 从 fillCenter 改为 fitCenter，两者保持一致

## GitHub Actions CI/CD
- 文件：`.github/workflows/android.yml`
- Debug 构建：每次 push 到 main 分支自动构建，APK 保存在 artifacts（7天）
- Release 构建：打 `v*.*.*` tag 时自动构建 release APK
- Release 发布：自动创建 GitHub Release 并上传 APK
- 签名：需在 Repo Settings → Secrets 添加 ANDROID_SIGNING_* 四个密钥
